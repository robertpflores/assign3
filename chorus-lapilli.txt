Chorus Lapilli Lab Notebook

Starting from my working tic-tac-toe in App.jsx. Need to turn it into
chorus lapilli — basically tic-tac-toe but after you place your 3 pieces
you have to move them around instead of placing more. Also there's a rule
about the center square.

Tracking moves
- got rid of the history array since we don't need time travel for this
- added a moveCount integer to Game that increments on every valid move
- xIsNext is just moveCount % 2 === 0
- to figure out which phase each player is in, I calculate how many moves
  they specifically have made:
  X moves = Math.ceil(moveCount / 2)
  O moves = Math.floor(moveCount / 2)
- once a player hits 3 moves, they switch to movement phase

Placement vs movement
- in handleClick, if the current player has < 3 pieces, it works like
  normal tic-tac-toe — click an empty square, place your symbol
- if they have 3 pieces, go to the movement logic instead

Two-click movement
- this was the trickiest part to get right
- added a selectedSquare state variable to Board, starts as null
- first click: player clicks one of their own pieces to select it
  - if they click an empty square or the opponent's piece, nothing happens
  - otherwise set selectedSquare to that index
- second click: player clicks where they want to move
  - immediately reset selectedSquare to null
  - check that the destination is empty
  - check that it's adjacent (wrote a helper for this)
  - if valid, do the move. if not, just cancel the whole thing
- at first I wasn't resetting selectedSquare before the checks, which caused
  some weird behavior where invalid moves would leave a piece "stuck" as
  selected. fixed by moving the null reset before validation

isAdjacent helper
- converts indices to row/col (row = Math.floor(i/3), col = i % 3)
- adjacent means rowDiff <= 1 and colDiff <= 1, but not the same square
- handles all 8 directions including diagonals

Center square rule
- if the current player has a piece on square 4 (center), their move has
  to either:
  1. vacate the center (i.e. from === 4), or
  2. win the game
- to check the win condition I create a copy of the board, apply the move
  on the copy, and run calculateWinner on it
- if neither condition is satisfied the move gets rejected

Preventing moves after win
- just check calculateWinner at the top of handleClick and return if
  there's already a winner
- this was already in the tic-tac-toe version so I just kept it

Testing
- played through a bunch of scenarios in the browser
- placement phase works fine, same as tic-tac-toe for the first 6 moves
- movement works: click your piece, click adjacent empty square, piece moves
- tried clicking non-adjacent squares — correctly rejected
- tried clicking occupied squares as destination — rejected
- tried moving opponent's piece — doesn't let you
- tested center rule: put O in center, then tried to move a different O piece
  without winning — blocked. moving the center piece out works. winning move
  from center also works.
- wrote some tests in test_chorus_lapilli.py to check the main things
